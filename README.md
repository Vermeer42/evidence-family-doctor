# 🩺 家庭医生 — 循证健康问答助手

> 基于权威医学指南的 AI 健康问答工具，为中国家庭提供可追溯、有证据等级的健康建议。

![首页](output/screenshot-home.png)

## 为什么做这个

市面上的健康问答产品存在一个空白：

- **面向医生的循证工具**（OpenEvidence、百川 M2）专业但普通人看不懂
- **面向普通人的 AI 问答**（蚂蚁阿福、腾讯健康）亲民但不标注证据来源

**家庭医生 = 循证严谨度 × 用户友好度**，每一条建议都标注来源和可信度。

## 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 RAG 证据检索 | 基于 BM25 从医学指南库中检索相关条目 |
| 📊 证据分级 | 四级标注（🟢强推荐 / 🟡一般建议 / 🔵专家共识 / 🔴仅供参考） |
| 🚨 就医红线 | AI 自主判断危险信号，强制提醒就医 |
| 📚 来源可追溯 | 每条回答附带原始指南出处 |
| 💬 流式对话 | SSE 实时流式输出，打字机效果 |

## 技术选型

| 层级 | 技术 | 选择原因 |
|------|------|----------|
| 前端 | 纯 HTML + Tailwind CSS | 轻量、无构建步骤、CDN 加载 |
| 后端 | Cloudflare Workers | 全球边缘部署、免费额度、国内可达 |
| 大模型 | Claude API (Sonnet) | 多模态、幻觉率低、中文能力强 |
| 检索 | BM25 + 同义词扩展 | 零成本零依赖、中文医疗术语覆盖好 |
| 知识库 | 结构化 JSON | 6 病种 29 条权威指南摘要 |

## 本地运行

```bash
# 1. 克隆仓库
git clone https://github.com/YOUR_USERNAME/evidence-family-doctor.git
cd evidence-family-doctor

# 2. 安装依赖
npm install

# 3. 配置 API Key
cp .dev.vars.example .dev.vars
# 编辑 .dev.vars 填入你的 Claude API Key

# 4. 启动后端（端口 8787）
npm run dev

# 5. 启动前端（另一个终端，端口 3000）
cd src/frontend && npx http-server -p 3000

# 6. 打开浏览器
# http://localhost:3000
```

## 设计决策

1. **为什么用 BM25 而非向量检索？** — 中转 API 不支持 embedding；29 条文档关键词检索已够用；零成本零延迟
2. **为什么前端不用 React/Vue？** — 3 个文件、零构建，面试官 clone 后能直接跑，降低理解门槛
3. **为什么证据等级用 emoji？** — 中老年用户认知成本最低，颜色+图标双通道信息传递
4. **就医红线为什么让 AI 判断而非关键词匹配？** — 关键词漏报率高，LLM 能理解语义上下文
