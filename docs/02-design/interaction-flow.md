# 交互流程 — 移动端优先设计

## 1. 页面架构（3页）

```
[首页] ──点击──→ [对话页] ──点击来源──→ [来源详情页]
```

## 2. 首页

### 2.1 布局

```
┌──────────────────────────────┐
│  🏥 家庭医生                  │  ← 顶部栏，24px 标题
│                              │
│  ┌────────────────────────┐  │
│  │ 有什么健康问题？          │  │  ← 输入区域
│  │                        │  │
│  │ [📷]  [🎤]        [发送] │  │  ← 操作按钮
│  └────────────────────────┘  │
│                              │
│  💡 你可以这样问：             │  ← 引导示例
│  ┌──────────┐ ┌──────────┐  │
│  │血压高要注意│ │体检报告看  │  │
│  │什么？     │ │不懂       │  │
│  └──────────┘ └──────────┘  │
│  ┌──────────┐ ┌──────────┐  │
│  │感冒了怎么│ │腰疼是什么  │  │
│  │办？      │ │原因？      │  │
│  └──────────┘ └──────────┘  │
│                              │
│  ──────── 历史对话 ────────  │
│  📋 昨天: 血压高吃什么药     │
│  📋 前天: 膝盖疼要不要拍片   │
└──────────────────────────────┘
```

### 2.2 关键设计决策

- **输入框突出**：上半区最大的交互元素，视觉焦点
- **字体大小**：正文 16px，输入框 18px，引导卡片 15px
- **引导卡片**：4 个常见场景示例，点击自动填入输入框
- **历史记录**：简单列表，最近的在前，点击恢复对话
- **MVP 不需要登录**（无状态，历史通过 localStorage 存储）

### 2.3 输入方式

| 方式 | 入口 | 行为 |
|------|------|------|
| 打字 | 点击输入框 | 标准键盘弹出，18px 字体方便阅读 |
| 语音 | 点击 🎤 按钮 | 浏览器语音识别 API → 文字转写 → 自动填入输入框，用户确认后发送 |
| 拍照 | 点击 📷 按钮 | 相机/相册选择 → 附加图片 → 用户可添加文字说明 → 发送 |

**语音输入详细流程：**
1. 点击 🎤 → 按钮变红，显示"正在听..."
2. 语音实时转文字填入输入框
3. 用户检查文字，可编辑修改
4. 点击"发送"提交（不自动发送——防止识别错误）

## 3. 对话页

### 3.1 布局

```
┌──────────────────────────────┐
│ ← 返回    家庭医生    [新对话] │  ← 导航栏
├──────────────────────────────┤
│                              │
│ ┌─ 用户 ────────────────┐   │
│ │ 我血压150/95，要吃药吗？ │   │
│ └───────────────────────┘   │
│                              │
│ ┌─ 🚨 就医红线 ──────────┐  │  ← 如触发
│ │ 请尽快就医...            │  │
│ └────────────────────────┘  │
│                              │
│ ┌─ AI ──────────────────┐   │
│ │ 🟡 本回答基于一般证据    │   │  ← 总等级卡片
│ │ 来源：中国高血压指南等   │   │
│ │ ─────────────────────  │   │
│ │ 根据最新指南，您的血压...│   │
│ │ ...                    │   │
│ │ 建议：                 │   │
│ │ • 限盐<5g/天 🟢        │   │  ← 内联等级标注
│ │ • 规律有氧运动 🟢       │   │
│ │ • 是否用药需医生评估 🟡  │   │
│ │ ─────────────────────  │   │
│ │ 📚 证据来源 (2)    ▼   │   │  ← 可折叠
│ └────────────────────────┘  │
│                              │
├──────────────────────────────┤
│ [📷] [输入健康问题...] [🎤][↑]│  ← 底部输入栏
└──────────────────────────────┘
```

### 3.2 回答渲染流程

```
用户发送消息
    │
    ▼
[加载中："正在查阅医学资料..." + 动态省略号]
    │
    ▼
[逐字流式输出回答内容]
    │
    ▼
[回答完成 → 渲染总等级卡片 + 来源脚注]
```

- 流式输出：逐 token 显示（打字机效果）
- 总等级卡片在回答完成后出现（需要完整上下文才能确定总等级）
- 内联标记随句子流式输出实时渲染

### 3.3 图片上传流程

```
用户点击 📷
    │
    ├─ 拍照
    │   └─ 拍摄 → 预览 → 确认
    │
    └─ 相册
        └─ 选择 → 预览 → 确认
            │
            ▼
[图片缩略图出现在聊天中 + 文字输入提示："请描述您的问题"]
            │
            ▼
用户添加说明（可选）→ 发送
            │
            ▼
AI 回答含图片分析 + 如为症状照片则一律标 🔴
```

### 3.4 交互细节

| 元素 | 点击行为 |
|------|---------|
| 内联等级标记（🟢🟡🔵🔴） | 底部浮层：显示来源名称 + 证据类型 |
| "📚 证据来源" | 展开/折叠来源列表 |
| 来源列表中的条目 | 跳转到来源详情页 |
| 红线横幅"查看附近医院" | 打开设备地图 |
| "新对话"按钮 | 清空上下文，重新开始（当前有内容时需确认） |

## 4. 来源详情页

### 4.1 布局

```
┌──────────────────────────────┐
│ ← 返回          证据来源详情  │
├──────────────────────────────┤
│                              │
│ 📖 中国高血压防治指南(2024)    │  ← 标题
│ ─────────────────────────── │
│ 发布机构：中华医学会           │
│ 证据等级：🟢 强证据           │
│ 类型：临床实践指南             │
│ ─────────────────────────── │
│                              │
│ 📋 相关原文摘录：             │
│ "对于1级高血压患者（收缩压    │
│  140-159mmHg），建议先进行    │
│  3-6个月生活方式干预..."      │
│                              │
│ ─────────────────────────── │
│ 🔗 查看原文                  │  ← 外部链接（如有）
│                              │
└──────────────────────────────┘
```

### 4.2 内容策略

- 展示实际被引用的 RAG 文本块（摘录，非全文）
- 来源元数据：发布机构、年份、证据类型
- 外部链接（部分指南有公开可访问的原文）
- 如来源为 PubMed 论文：显示标题、作者、期刊、DOI 链接

## 5. 适老化设计

### 5.1 字体与触控

| 元素 | 尺寸 | 触控区域 |
|------|------|---------|
| 正文 | 最小 16px | — |
| 输入框文字 | 18px | — |
| 按钮 | 标签 16px | 最小 48×48px |
| 引导卡片 | 15px | 最小高度 44px |
| 等级标记 | 图标 12px + 提示 14px | 点击区 36×36px |

### 5.2 色彩与对比度

- 所有文字：对比度 ≥ 4.5:1（WCAG AA 标准）
- 等级颜色配合图标+文字使用（不单独依赖颜色），兼顾色盲用户
- 深色模式：MVP 不做（降低复杂度，中老年用户很少使用）

### 5.3 交互简洁性

- 任何功能最多 2 步到达
- 不依赖滑动手势（所有操作通过可见按钮完成）
- 历史记录不使用无限滚动——分页显示（每页 20 条）
- 明确的"返回"按钮，不依赖系统返回手势
- 错误提示用大白话："网络不好，请稍后再试"，绝不显示技术报错

## 6. 响应式适配

| 屏幕宽度 | 布局 |
|---------|------|
| < 480px（手机） | 默认，单列布局 |
| 480-768px（大屏手机/小平板） | 相同布局，略宽边距 |
| > 768px（平板/电脑） | 居中 480px 最大宽度内容列，字体 +2px |

MVP 以移动端为主；桌面端可用但不专门优化。
