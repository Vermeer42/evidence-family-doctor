# 竞品调研报告

> 调研时间：2026-06-15
> 调研方法：桌面调研（Google/知乎/行业报道联网检索）
> 目的：明确市场格局、找准产品差异化定位

---

## 一、市场全景：谁在做"AI + 健康问答"

当前市场可分为 **四个赛道**，按目标用户和循证深度划分：

```
                    面向医生 ←————————————→ 面向患者/普通人
                        │                        │
  循证深度高    OpenEvidence │ 百川M2  证元芳     │  （空白地带）← 你的机会
                    氢离子（阿里健康）            │
                        │                        │
  循证深度低            │  通用大模型             │  蚂蚁阿福  丁香AI
                        │  （文心/通义/DeepSeek） │  腾讯健康
                        │                        │  平安好医生
```

**关键发现：高循证 + 面向普通人 = 目前几乎没人做。** 现有循证产品全部面向医生，普通人用的产品循证深度都不够。

## 二、竞品详细分析

### 2.1 国外标杆：OpenEvidence

| 维度 | 详情 |
|------|------|
| 定位 | 面向医生的循证临床决策支持 |
| 规模 | 覆盖美国40%+医生，$3.5B估值，融资$210M |
| 核心能力 | 实时综合医学文献，提供循证问答，USMLE满分 |
| 证据处理 | 综合+可视化临床证据，追溯到原始文献 |
| 用户群 | 仅面向医疗专业人员，免费使用 |
| 产品形态 | Web + App |
| **与你的关系** | **你的"证据呈现"可参考它的设计，但你面向普通人，语言和交互要完全不同** |

> 注：2026年4月 OpenAI 推出 ChatGPT for Clinicians 正面竞争 OpenEvidence。这说明"循证AI"赛道在爆发。

### 2.2 国内循证赛道（面向医生）

#### 百川 Baichuan-M2 Plus
| 维度 | 详情 |
|------|------|
| 定位 | 首个循证增强医疗大模型 |
| 核心创新 | "六源循证推理范式"——证据分层 + PICO检索 + 动态更新 |
| 证据处理 | 优先采纳高质量信息（RCT > 观察研究），内置证据评估器 |
| 幻觉率 | 比 DeepSeek-R1 低 3 倍 |
| 用户群 | ToB（医院、医疗机构） |
| **与你的关系** | **技术理念最接近你——证据分层。但它是模型层的能力，你是应用层RAG+prompt实现，路径不同** |

#### 阿里健康-氢离子
| 维度 | 详情 |
|------|------|
| 定位 | 医生版GPT，循证决策支持 |
| 核心创新 | "动态证据定位"功能，独家接入BMJ顶刊 |
| 证据处理 | 标注依据来源（指南原文），可点击查看 |
| 用户群 | 医生 |
| **与你的关系** | **"点击查看指南原文"的交互设计值得你参考，但要翻译成普通人能懂的语言** |

#### 证元芳（轻松健康集团）
| 维度 | 详情 |
|------|------|
| 定位 | 循证医学智能体，通过信通院MedClaw测评 |
| 核心创新 | 从底层架构即内嵌循证体系，"证据优先、来源可溯"为原生设计原则 |
| 证据处理 | 每次回答标注证据层级 + 结论可验证性 |
| 用户群 | 医生 + 部分患者端（但主打B端） |
| **与你的关系** | **理念最相似的国内竞品，但定位偏B端医生工具。你做C端普通人** |

### 2.3 面向普通用户的产品

#### 蚂蚁阿福（蚂蚁集团）
| 维度 | 详情 |
|------|------|
| 定位 | AI健康朋友，每天回答500万+健康问题 |
| 规模 | 1500万用户，55%来自三线及以下城市 |
| 核心功能 | 健康问答、症状自查、报告解读、挂号咨询 |
| 证据处理 | ❌ 未明确标注证据来源和分级 |
| 安全机制 | 每条回答尾部标注"AI生成仅供参考"；近期试点"医生把关"功能 |
| 适老化 | 有，面向下沉市场 |
| **与你的关系** | **用户场景最像（中老年、下沉市场），但它不做证据分级——这正是你的差异化** |

#### 腾讯健康 AI 智能体
| 维度 | 详情 |
|------|------|
| 定位 | 基于混元大模型的健康问答 |
| 核心功能 | 健康自诊、用药指导、报告解读 |
| 证据处理 | 整合"权威知识库"，但用户端不可见证据层级 |
| **与你的关系** | **大厂基础设施，但不透明，用户无法验证来源** |

#### 丁香医生 AI
| 维度 | 详情 |
|------|------|
| 定位 | 基于丁香园医学内容库的AI问答 |
| 核心能力 | 内容质量较高（编辑审核），但AI层面创新有限 |
| 证据处理 | 依赖丁香园已有内容库，非实时循证检索 |
| **与你的关系** | **内容质量可参考，但它是"编辑模式"不是"循证模式"** |

## 三、竞品空白与你的机会

### 市场空白总结

| 能力维度 | OpenEvidence | 百川M2 | 证元芳 | 蚂蚁阿福 | **你的产品** |
|----------|:-:|:-:|:-:|:-:|:-:|
| 面向普通人 | ❌ | ❌ | ❌ | ✅ | ✅ |
| 证据分级可见 | ✅ | ✅ | ✅ | ❌ | ✅ |
| 来源可追溯 | ✅ | ✅ | ✅ | ❌ | ✅ |
| 中文权威知识库 | ❌ | ✅ | ✅ | 不透明 | ✅ |
| 适老化设计 | ❌ | ❌ | ❌ | 部分 | ✅ |
| 就医红线兜底 | N/A | 未知 | 部分 | 弱 | ✅ |
| 多模态(图片) | ❌ | 未知 | ❌ | 部分 | ✅ |

**你的产品填补的空白：把"高循证标准"（现在只有面向医生的产品在做）带给普通用户，且做到适老化。**

### 差异化定位一句话

> 蚂蚁阿福让普通人能问AI健康问题，但你看不到答案来自哪里、有多靠谱；
> OpenEvidence让医生看到完整证据链，但普通人根本用不了。
> **你的产品 = OpenEvidence的循证严谨度 × 蚂蚁阿福的用户友好度。**

## 四、从竞品中可以借鉴什么

### 产品设计层面
1. **蚂蚁阿福** → 用户场景、适老化交互、"AI+医生把关"的安全思路
2. **OpenEvidence** → 证据呈现方式、引用格式、可信度标注
3. **证元芳** → 证据层级的分类体系（指南 > 系统综述 > 单篇研究）
4. **氢离子** → "点击查看指南原文"的交互

### 技术层面
5. **百川M2** → 证据分层融入推理链的思路（你在prompt层实现类似效果）
6. **蚂蚁阿福** → 每条回答强制加免责声明的合规做法

### 避免的坑
7. 不要像通用大模型那样"什么都答"——**做窄做深**（先聚焦高频场景）
8. 不要像蚂蚁阿福那样证据来源不透明——**透明是信任的基础**
9. 不要像OpenEvidence那样全专业术语——**翻译成人话**

---

## 五、对后续产品设计的启示

1. **核心功能优先级调整**：证据分级的"可视化呈现"要做成产品核心体验，不是藏在角落里的附属信息
2. **安全机制参考蚂蚁阿福**：免责声明 + 就医红线 + 不诊断只提供信息
3. **知识库策略**：先做3-5部国家指南（覆盖三高、糖尿病、常见症状），跑通证据链路
4. **PM故事加分点**：你能说"我研究了OpenEvidence（$3.5B独角兽）和百川M2的循证方法论，把它们的核心思路带到C端产品"
