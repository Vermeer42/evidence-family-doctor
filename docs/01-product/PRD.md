# 产品需求文档（PRD）：中国人的家庭医生

> 版本：v1.0（MVP）
> 日期：2026-06-15
> 状态：已定稿，待开发

---

## 一、产品概述

### 1.1 一句话定义
基于权威医学证据的健康问答助手，为中国中老年家庭提供可信、分级、安全的健康信息。

### 1.2 解决的问题
中国缺少欧美普遍存在的家庭医生制度，中老年人的健康信息主要来自营销号、搜索引擎和不可靠的网络内容，错误信息导致延误就医或过度焦虑。

### 1.3 产品价值主张
> 把医生级别的循证医学标准（目前只有面向专业人士的产品在做），用普通人能听懂的方式呈现，并在关键时刻引导就医。

### 1.4 目标用户
- **核心用户**：50岁以上中老年人，有健康疑问，缺乏专业辨别能力
- **次要用户**：替父母操作/转发的子女（25-40岁）

---

## 二、设计原则

| 原则 | 含义 |
|------|------|
| 证据优先 | 每个回答都基于RAG知识库的权威来源，不靠模型记忆编造 |
| 透明可信 | 用户能看到答案来自哪里、有多可靠 |
| 安全兜底 | 识别危险信号时强制引导就医，不诊断 |
| 适老友好 | 大字体、简交互、说人话，不用专业术语 |
| 做窄做深 | MVP聚焦高频场景，不追求大而全 |

---

## 三、核心功能：证据分级体系

### 3.1 四级证据强度

| 等级 | 标识 | 含义 | 对应证据类型 |
|------|------|------|------|
| 强推荐 | 🟢 | 多部权威指南一致推荐，可放心遵循 | 国家/学会临床指南、多项系统综述一致 |
| 一般建议 | 🟡 | 有较好研究支持，但需结合个人情况 | 单部指南、系统综述、随机对照试验(RCT) |
| 专家共识 | 🔵 | 专家普遍认同，但缺乏强研究证据 | 专家共识、观察性研究 |
| 仅供参考 | 🔴 | 证据不足或存在争议，谨慎对待 | 单篇小研究、个案、机制推测 |

### 3.2 呈现规则
- 每条具体建议前置等级标识
- 每个回答末尾列出**来源卡片**：来源名称 + 类型（指南/综述/研究）+ 可点击查看
- 症状照片识别结果**一律标 🔴**，强制附"需面诊"

---

## 四、核心功能：就医红线机制

### 4.1 触发后产品行为
**依然给出信息，但页面顶部显示大红色横幅强制提醒"请尽快就医"。** 不拒绝回答，但把就医提醒放在最显眼位置。

### 4.2 红线触发条件（示例，详见 red-flags.md）
- 急性胸痛、呼吸困难、剧烈头痛
- 异常出血、意识改变、肢体无力
- 血压/血糖等关键指标处于危险区间
- 症状描述中出现急症关键词

### 4.3 红色横幅内容
```
⚠️ 这种情况建议尽快就医
根据你描述的情况，建议尽快到【XX科】就诊或拨打120。
以下信息仅供了解，不能替代医生面诊。
```

## 五、MVP 功能清单

### 5.1 纳入 MVP（v1.0）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 文字健康问答 | 输入问题→基于RAG的循证回答+证据分级 | P0 |
| 体检报告解读 | 上传报告图片→解读指标+建议+分级 | P0 |
| 就医红线提醒 | 危险信号→顶部红色横幅 | P0 |
| 症状照片识别 | 上传症状照片→描述特征+就医方向（**全部🔴+强制面诊**） | P1 |
| 来源引用展示 | 回答附权威来源卡片 | P0 |
| 流式输出 | 回答逐字显示 | P1 |

### 5.2 不纳入 MVP（后续版本）

| 功能 | 原因 | 计划版本 |
|------|------|--------|
| 用药相互作用查询 | 需专门药物数据库 | v2.0 |
| 健康档案/历史记录 | 需账号体系+存储 | v2.0 |
| 家庭成员管理 | 需账号体系 | v2.0 |
| 真人医生把关 | 需医生资源对接 | v3.0 |
| 挂号/就医服务 | 需对接医疗机构 | v3.0 |

## 六、信息架构与页面流程

### 6.1 页面结构（极简，3个核心页面）
```
首页（对话入口）
  ├── 输入框（文字提问）
  ├── 拍照/上传按钮（报告/症状照片）
  └── 常见问题快捷入口（三高、体检、用药...）
        ↓
对话页（问答主界面）
  ├── 用户提问气泡
  ├── AI回答（含证据分级标识）
  ├── [红线触发] 顶部红色横幅
  └── 来源卡片区
        ↓
来源详情（点击来源卡片）
  └── 展示指南/研究的具体出处和原文片段
```

### 6.2 核心交互流程
```
用户提问
  ↓
[后端] RAG检索权威知识库
  ↓
[后端] 红线规则判断 → 触发？→ 是：标记需就医
  ↓
[Claude] 基于检索证据生成回答 + 标注分级
  ↓
[前端] 渲染回答 + 分级标识 + 来源卡片 + (红线横幅)
```

## 七、非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 首字响应 < 3秒（流式） |
| 可用性 | 用户无需翻墙、无需注册即可使用 |
| 适老化 | 默认大字体（≥16px），高对比度，操作步骤≤2步 |
| 合规 | 全程标注"仅供参考，不替代就医"；不做诊断 |
| 隐私 | 不存储用户健康数据（MVP阶段无账号） |
| 成本 | demo阶段控制在API免费/低成本额度内 |

---

## 八、成功指标（demo阶段）

由于是 demo，不追求商业指标，关注**质量验证**：
- 回答的证据引用准确率（人工评估）
- 就医红线触发的准确率（安全压测）
- 父母真实试用的可用性反馈
- 与通用大模型对比的回答质量差异

---

## 九、风险与边界

| 风险 | 应对 |
|------|------|
| 用户当成诊断延误就医 | 全程免责声明 + 红线兜底 |
| RAG检索到矛盾信息 | 呈现时标注分歧，倾向高等级证据 |
| 症状照片误判 | 全部标🔴 + 强制面诊提醒 |
| 知识库覆盖不全 | 检索不到时明确告知"该问题缺乏权威依据，建议咨询医生" |
| 医疗合规 | 定位为信息工具非诊断工具，demo不商用 |
